## 📂 Directory Structure

Each recipe must include the following structure and files:

```
<recipe-name>/
├── pyproject.toml           # Python package/dependency configuration (Python only)
├── uv.lock                  # Dependency lockfile generated by uv (required)
├── manifest.yaml            # Metadata and ownership details (validated by CI)
├── README.md                # User-facing doc: what it does, how to run, features (required)
├── .env.example             # Environment variable declarations with placeholder values (no real secrets!)
├── AGENTS.md                # Architecture details used/generated by agents-cli (optional)
├── app/                     # Agent source code (recommended, but not required)
│   ├── __init__.py          # Package entry point — bootstraps load_dotenv() here
│   └── agent.py             # Agent definition — must define root_agent and app
└── tests/                   # Test suite
    ├── test_runnability.py  # Required runnability check
    ├── integration/         # Optional integration tests
    └── eval/                # Optional evaluation test sets
```

The individual files are covered in more detail in the sections and pages
below:

*   `manifest.yaml` — see the dedicated [Manifest](manifest.md) page.
*   `.env.example` and the `load_dotenv()` bootstrap in `__init__.py` — see [Environment Variables](environment-variables.md).
*   `tests/test_runnability.py` — see [Testing](testing.md).

## Dependency Management (`pyproject.toml`)

All Python recipes must use `pyproject.toml` managed via **`uv`**. **Do not use `requirements.txt` or `pip`.**

### Example `pyproject.toml`
Here is a baseline, minimal `pyproject.toml` template that you can copy and customize for your new Python recipe:

```toml
[project]
name = "your-recipe-name"
version = "0.1.0"
requires-python = ">=3.11,<3.14"
dependencies = [
    "google-adk[gcp]>=2.0.0,<3.0.0",
    "python-dotenv>=1.0.0",
]

[dependency-groups]
dev = [
    "pytest>=8.3.4",
]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["app"]

[tool.pytest.ini_options]
pythonpath = "."
# Integration tests require live credentials and are excluded from CI.
# Run them locally with: uv run pytest tests/integration
addopts = "--ignore=tests/integration"
```

### Generating `uv.lock`
`uv.lock` is a required file but is **not** hand-written. After editing
dependencies, generate or refresh it from the recipe folder so it stays in sync:
```bash
uv lock
```

## Recipe Documentation (`README.md`)

Every recipe must include a `README.md` at its root to guide users on design, prerequisites, environment setup, and running instructions.

### Template for `README.md`
Here is a standard, clean `README.md` layout. Copy and customize this for your recipe:

````markdown
# [Recipe Display Name]

A short, one-to-two sentence description of what the agent/recipe does, the use-case it addresses, and the primary ADK patterns or LLM capabilities it demonstrates.

## 📋 Agent Details

Provide some details about the recipe so users can quickly understand its scope, target architecture, and patterns.

## ⚙️ Getting Started

### 1. Prerequisites
Explain any external tools, APIs, or software required (e.g., `gcloud CLI`, specific GCP permissions, or local API accounts).

### 2. Setup Environment Variables
Before running the agent, you must copy the `.env.example` file to create a local `.env` configuration file:
```bash
cp .env.example .env
```
Open the newly created `.env` file and manually update all placeholder variables with your actual credentials, API keys, GCP Project ID, or model preferences.

### 3. Install Dependencies
Run the standard installation command inside this recipe's folder:
```bash
uv sync
```

### 4. Authentication & Setup
Perform any necessary cloud provider authentication or environment-specific configuration steps required for the agent to access external APIs and services.

## 🚀 Running the Agent

Start the agent locally in interactive mode:
```bash
uv run adk run app
```

## 🧪 Testing

To run the local unit and runnability checks:
```bash
uv run pytest
```

> **Note:** Tests under `tests/integration/` require live credentials and are excluded from CI automatically. To run them locally: `uv run pytest tests/integration`
````

## Non-Python Recipes

The `pyproject.toml`, `uv.lock`, and `tests/test_runnability.py` requirements
above are specific to **Python** recipes. Recipes in other supported languages
(`java`, `go`, `kotlin`, `typescript`) follow their own ecosystem's conventions
for dependency management and testing. Regardless of language, **every** recipe
must still include:

*   `manifest.yaml` (with `language` set to the correct value) — see [Manifest](manifest.md).
*   `README.md`
*   `.env.example` (whenever the recipe reads environment variables) — see [Environment Variables](environment-variables.md).

The size and directory-naming constraints in [Design & Naming](design-and-naming.md) apply to recipes in all languages.
